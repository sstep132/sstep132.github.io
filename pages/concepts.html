Important Concepts to Understand

  1. Client - A web browser or other device or program that makes requests for data or services.
  2. Server - The system that responds to a client request with data and resources that it manages.
  3. Communication Between and Client-Server - a client requests information from a server and the server sends the information back across a network.
  4. Requests and Responses - a client requests a piece of data, a web page, or a file in message form.
  5. Legacy Systems - older, outdated systems that are using old technology.
  6. Status Codes - introduced with HTTP/1.0 provide status information about a request to give the client information about what happened.
  7. Pipelining - the ability to send multiple requests without the need for a response on the same socket.
  8. Multiplexing - the ability to send use a single TCP connection that allows multiple requests at the same time.
  9. The Statelessness of HTTP - every request is treated as separate by the server without automatically storing any memory from prior requests.
  10. The Evolution of HTTP - HTTP/0.9 as a single-line protocol that was able to send GET requests, HTTP/1.0 brought headers, status codes, and support for different file types, HTTP/1.1 allowed for standardization, the reuse of a TCP connection, and pipelining.
  11. Future of HTTP/3 and QUIC adoption - QUIC uses UPD to run multiple streams that are independent from each other so that a single stream can be blocked and data can continue to transfer if there is packet loss.
