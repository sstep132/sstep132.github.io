Important Concepts to Understand

  1. Communication Between and Client-Server - a browser, also called a client requests information from a server and the server sends the information back across a network.
  2. Requests and Responses - a client requests a piece of data, a web page, or a file in message form.
  3. The Statelessness of HTTP - every request is treated as separate by the server without automatically storing any memory from prior requests.
  4. The Evolution of HTTP - HTTP/0.9 as a single-line protocol that was able to send GET requests, HTTP/1.0 brought headers, status codes, and support for different file types, HTTP/1.1 allowed for standardization, the reuse of a TCP connection, and pipelining.
  5. Future of HTTP/3 and QUIC adoption - QUIC uses UPD to run multiple streams that are independent from each other so that a single stream can be blocked and data can continue to transfer if there is packet loss.
