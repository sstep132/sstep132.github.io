A Study of HTTP

Introduction

There are millions of people who use the internet daily to send and receive data at a near-instant rate. What they don’t realize is that browsers and routers don’t actually transfer data instantly or magically. Behind the scenes, there is a set of rules that are followed by devices that allow communication between them to send and receive information. That is why the Hypertext Transfer Protocol, or HTTP, was invented, and without it, communication between devices would be too complicated since each device would be communicating differently.This website explains what HTTP is, why it was created, how it works, how it has evolved into the protocol that powers today's web, and its future.

Key Terms

  • Hypertext Transfer Protocol(HTTP) - a client-server protocol that operates on the application layer on top of the Transmission Control Protocol (TCP) so that a client can communicate with a server to make a request for and receive a resource successfully.
  • Tim Berners-Lee - a computer science pioneer who was born in England and invented HTTP.
  • Client-server architecture - a client, such as a web browser, sends requests to a server who processes the requests and returns the requested data to the client.
  • Transmission Control Protocol(TCP) - an internet protocol that makes reliable communication possible between devices on a network and is used in versions of HTTP prior to HTTP/3.
  • Hypertext Markup Language(HTML) - a hypertext markup language that allows the browser to understand exactly how the web page or information should be displayed to the user.
  • QUIC - a transport protocol which is on top of UDP and is the building block of HTTP/3.
  • Latency - the amount of time between when an action is initiated and when the client recieves the response from the server.
